리눅스 자격증 통합 준비 과정의 50p 문제
계정 설정의 연습문제
사용자 www-hacker, www-user, data-user 그리고 db-user를 생성한다.
모든 사용자는 비밀번호를 rockylinuxworld라고 설정한다.
사용자 www-hacker는 웹 서비스에 접근은 가능하나, 시스템에 접근은 불가능하다.
www-user라는 그룹에 포함이 된다.
www-user,data-user는 secure-data그룹에 포함이 된다.
각 사용자는 tcsh를 통해서 시스템에 접근이 가능하다.
db-user는 wheel그룹에 포함이 되며, 시스템에 로그인이 가능하다.
각각 사용자를 ssh로 접근 후 "id"명령어 실행한다.

chgrp : 그룹 권한만 지원
chgrp를 사용하는 사용자가 해당 파일의 소유권을 가지고 있어야 하고, 변경하려는 그룹에 그룹원으로 가입을 하고 있어야 한다.

-rw-r--r--. 1 root root 0 Jun 12 14:21 file1

rw-r--r-- -> 권한의 표시
rwxrwxrwx > 반복 되어 있다.
rwx|rwx|rwx
[U] [G] [O]
U user : 사용자
G group : 그룹사용자
O other : 전체 - {사용자, 그룹사용자}

find의 옵션은 보통 인지값을 가지고 있기 때문에 옵션과 해당 인자를 붙여서 잘 사용하는것이 중요하다.

-name : 파일 이름을 기준으로 검색
-iname: 대소문자를 구별하지 않고 파일 이름을 기준으로 검색
-perm : 파일 권한을 기준으로 검색한다.

find . -perm [퍼미션]
600    : 정확히 rw-------를 찾는다.
-600   : U자리의 rw를 찾고 나머지 권한 자리는 있든 없든 검색

find에서 perm은 해당 자리의 권한을 포함한 나머지를 찾는것이 많기 때문에 -를 입력하는 경우가 대부분이다.

-type : 파일의 종류를 기준으로 검색
b : 블록 파일
c : 문자
d : 디렉토리
f : 파일
l : 링크
s : 소켓

-size : 파일의 크기를 기준으로 검색
+n : n보다 크다
-n : n보다 작다
n : n이다
b : 512-byte
c : Bytes (Character = Byte)
k : Kilo Byte
M : Mega Bytes
G : Giga Bytes
w : 2-byte

-links : 링크의 개수를 기준으로 검색
-user : 사용자 ID를 기준으로 검색
-group: 그룹 ID를 기준으로 검색
-atime : 특정 기간 이상 접근하지 않은 파일을 기준으로 검색
-mtime : 특정 기간 이상 수정되지 않은 파일을 기준으로 검색
-inum : number 지정된 inode 번호와 파일을 찾는다.
-print : 표준출력으로 검색된 파일 출력: GNU는 디펄트, Unix는 필수 입력
-exec command {} ₩; :찾은 각 파일에 대해 지정된 명령을 실행
-ok command {} \; : 실행여부(실행 되어 있는지 아닌지)를 사용자에게 확인 후 명령을 실행

find 명령어에서 -exec 옵션에서 중괄호 {}의 역할
find 명령어는 검색으로 출력되는 하나 이상의 파일 및 디렉토리 결과를 가지고 있다. 이 결과를 바탕으로 -exec 옵션에서는 2차적으로 명령을 진행가능하다.
이때 {}는 버퍼의 역할을 하며 순차적으로 처음 검색된 내용부터 차례대로 치환된다. 이 치환은 마지막 검색내용이 입력될때까지 반복된다.

find 명령어에서 -exec 옵션에서 세미콜론(;)의 역할
find 명령어를 사용하면서 출력된 결과를 가지고 바로 후처리 실행으로 이어질수 있는 옵션인 exec ok가 존재한다. 해당 문법의마지막에는 세미콜론(;)이 입력된다. 하지만 쉘에서는 세미콜론(;)을 기준으로 앞 명령어와 뒤 명령어를 구분하는 구분자로 사용하기 때문에 그냥 사용할수는 없으며 바로 앞에 역슬래시를 붙여 \; 구조로 만들어줘야 find 명령어에서 정상적으로 해석가능하다.

student가 소유한 모든 파일을 찾아 /root/student.list 디렉토리에 파일을 복사
# find / -user student -exec cp -fv {} /root/student.list \;

# find / -user student -type f -exec cp -fv {} /root/student.list \;

네트워크 매니저 :

네트워크 설정을 위한 기본 정보
IP: 엔드디바이스를 위한 주소
공인망 / 사설망 주소로 나뉜다.
IPv4* / IPv6 
192.168.0.5  172.16.15.4
netmask:호스트가 어느 구역에 속해 있는지를 파악하기 위해 필요
gateway:LAN 통신이 아닌 다른 구역으로 통신을 하기 위해서는 게이트웨이가 필요하다. 
dns: DOMAIN -> IP / IP -> DOMAIN 질의 서비스를 하기 위해 필요함. /etc/resolv.conf

현재 버전에서는 /etc/sysconfig/network-scripts를 사용하지 않고
/etc/NetworkManager/system-connections/ 디렉토리를 사용한다.

하나의 NIC에는 여러개의 프로파일이 존재 할 수 있음.
NIC에는 한번에 하나의 프로파일만 적용 가능
