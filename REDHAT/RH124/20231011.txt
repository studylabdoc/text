방향 재지정자:
[root@servera 11238]# cd fd
fd = file discripter : 파일 기술자. 

c언어 stdio.h : 입출력에 대한 헤더 파일..
[root@servera fd]# ls -l
total 0
lrwx------. 1 root root 64 Oct 12 06:16 0 -> /dev/pts/1
lrwx------. 1 root root 64 Oct 12 06:16 1 -> /dev/pts/1
lrwx------. 1 root root 64 Oct 12 06:16 2 -> /dev/pts/1

한개의 파일을 열게 되면 파일기술자가 쉘에 의해 생성
연결을 나타내는 특별한 숫자들이 있다.
그중에 0 1 2번은 예약되어 있음.
이 번호들에 대하여 알아야 함.
이 번호들은 이미 예약되어 있기 때문에 파일을 열때 따로 주어지지 않음.

0: stdin : 표준 입력 : 따로 지정이 없다면 키보드로 입력
1: stdout  : 표준 출력 : 따로 지정이 없다면 터미널로 출력
2: stderr : 표준 에러 : 따로 지정이 없다면 터미널로 출력
 stdin ---------- > BASH    ---------->1: stdout 
                                      ----------> 2: stderr

기본적으로 사용할 때는 지금처럼 키보드에 입력하고 모니터로 출력하는 형태를 계속 사용

표준 출력 리다이렉션

# cmd > file  덮어쓰기
# cmd >> file1 이어쓰기

# cmd 1> file  덮어쓰기
# cmd 1>> file1 이어쓰기
기본이 1번이기 때문에 생략가능
에러
# cmd 2> file  덮어쓰기
# cmd 2>> file1 이어쓰기
번호를 생략할 수 없다.
