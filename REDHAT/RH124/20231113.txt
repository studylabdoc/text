inode 번호가 틀림 : 파일 자체가 다른것
파일의 형식 : l > 링크파일
권한 부분 : 전체 모든 권한이 있다.
해당 권한을 조작해도 의미가 없다.
원본 파일의 권한으로 동작.

file3 -> file1 : file3을 액세스 하면 file1에 액세스 하는 것
* inode로 표시되지 않고 파일 명

:q : 빠져나가기

:wq!
:w!
강제의 의미로 !룔 붙여준다.

검색
/패턴
/\c패턴 : 대소문자 구별 없이 검색
 
쉘 명령어 실행
:!CMD

명령어를 찾는 순서.

1.  hash 에서 사용 [메모리상에 등록되어 있는 정보를 가지고 실행]
삭제할 경우 -> hash -r
2. alias 에서 명령어 있는지 확인
3. $PATH 변수에서 명령 검색
/root/.local/bin
/root/bin
/usr/local/sbin
/usr/local/bin
/usr/sbin
/usr/bin
path 변수에서는 처음 등록된 디렉토리부터 찾아 나가며 명령어가 찾아지는 즉시 실행.
